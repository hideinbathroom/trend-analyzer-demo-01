# 📊 Trend Analyzer Demo

실시간 주식 시장 데이터를 수집하고, LLM으로 분석하여 Threads / Discord / Telegram에 자동 브리핑을 게시하는 봇입니다.

> 뉴스 수집 → 속보 감지 → LLM 시장 분석 → 멀티 채널 자동 게시

## 🎯 주요 기능

### 1. 뉴스 수집 & 속보 감지 시스템
- RSS 피드 9개 소스 + 네이버 뉴스 API로 실시간 뉴스 수집
- 9차원 다차원 스코어링으로 뉴스 중요도 자동 평가
- 자카드 유사도 기반 주제 클러스터링 + Velocity Signal로 속보 자동 감지
- 지수 감쇠 함수(`exp(-λ × age)`)로 신선도 반영

### 2. 거시경제 데이터 수집
- Yahoo Finance: 미국 3대 지수, 선물, VIX, 환율, 원자재, 국채, 비트코인
- 네이버 금융: 코스피/코스닥 실시간 크롤링 (yfinance 폴백)
- CNN: 공포탐욕지수 (API → 크롤링 → VIX 기반 추정 3단계 폴백)
- 유럽 지수: 유로스톡스50, DAX

### 3. LLM 기반 2단계 분석 & 브리핑 생성
```
데이터 수집 → [1차 LLM] 시장 정세 판단 → [2차 LLM] 브리핑 생성 → 게시
```
- 1차: 매크로 전략가 관점에서 시장 국면/리스크/기회 분석
- 2차: 분석 결과 + 원본 데이터로 500자 이내 브리핑 생성

### 4. 트럼프 Truth Social 모니터
- CNN 아카이브에서 트럼프 최신 글 수집 (Range 헤더로 64KB만 요청)
- LLM이 한국 주식시장 영향도 판단 (HIGH/LOW)
- HIGH 판정 시 자동으로 시장 영향 분석 포스트 생성 & 게시

### 5. 속보 자동 게시
- 10분 간격 폴링으로 고영향 속보 감지 (score ≥ 20)
- 자카드 유사도 기반 중복 방지
- 일일 3건 한도 + 15분 쿨다운

### 6. 멀티 채널 동시 게시
- **Threads**: Meta Graph API (본문 + 테마 댓글 분리)
- **Discord**: Embed 카드 형식 (뉴스 헤드라인 + 브리핑)
- **Telegram**: Bot API 메시지 전송

## 📊 브리핑 시스템 (모닝 / 장 마감)

2단계 LLM 호출 구조로 브리핑을 생성합니다.

### 모닝 브리핑 (08:00, 장 전 전용 데이터)

```
장 전 전용 데이터 수집 (fetch_premarket_data)
  미국 3대 지수 전일 종가 (나스닥, S&P500, 다우존스)
  미국 선물 (S&P500선물, 나스닥선물) ← 오늘 장 방향성 핵심
  유럽 전일 종가 (유로스톡스50, DAX)
  한국 전일 종가 (코스피, 코스닥) ← "전일" 명시
  야간 지표 (달러/원, VIX, WTI, 금, 비트코인, 미국10년물국채)
  공포탐욕지수 (CNN API → VIX 기반 추정)
  + 뉴스 수집 (fetch_news → 점수화 → 상위 6건 LLM 전달)
    → [1차 LLM] 시장 정세 판단
      → [2차 LLM] 장 전 관점 브리핑 생성
        → Threads + Discord + Telegram 게시
```

모닝 브리핑은 장 시작 전이므로:
- 한국 지수는 "전일 종가"임을 명시 (어제 데이터를 오늘 것처럼 보여주지 않음)
- 미국 선물/야간 지표 기반으로 오늘 장 방향성을 전망

#### 모닝 브리핑 포맷 (번호 리스트 10항목)
```
{날짜} - 어제 밤부터 아침사이에 일어난 일

1. 나스닥 ▼1.2%, S&P500 ▼0.8%
2. 원유 98달러 ▲2.8%, 환율 1,505원
3~8. 주요 뉴스 6건 (각 25자 이내)
9. 오늘 장 전망 한줄
10. 팔로우&하트로 매일 속보와 브리핑 확인
```

### 장 마감 리포트 (16:50)

```
실시간 데이터 수집 (fetch_macro_data)
  + 뉴스 수집 (fetch_news → 점수화 → 상위 6건 LLM 전달)
  → [1차 LLM] 시장 정세 판단
    → [2차 LLM] 정세 판단 + 원본 데이터 → 최종 브리핑 생성
      → 500자 초과 시 LLM에게 축약 재요청
        → 게시 (본문 + ---THEME--- 구분자로 테마 댓글 분리)
```

#### 장 마감 브리핑 포맷 (번호 리스트 10항목)
```
{날짜} - 오늘 주식 시장 일어난 일
[한줄요약 25자 이내]

1. 코스피 지수 + 등락률
2. 코스닥 + 환율 또는 수급
3~8. 주요 뉴스 6건 (각 25자 이내)
9. 내일 전망 한줄
10. 팔로우&하트로 매일 속보와 브리핑 확인
```

- 각 항목 25자 이내 (Threads 모바일 한 줄에 딱 맞게)
- 주도 테마는 본문이 아닌 reply 체인으로 자동 게시
- 소수점 숫자는 Threads 스팸 필터 방지를 위해 정수로 자동 변환
- 본문/테마 각각 500자 제한

## 🔧 핵심 메커니즘

### 뉴스 수집 & 스코어링 파이프라인

![뉴스 파이프라인](docs/news-pipeline.png)

### 뉴스 스코어링 (9차원)

| 차원 | 설명 | 범위 |
|------|------|------|
| 영향도 (impact) | 키워드 사전 매칭 (매크로/지정학/수급/기업/섹터) | 0~15+ |
| 출처 신뢰도 | 연합뉴스/한경=3, 매경/머니투데이=2, 기타=1 | 1~3 |
| 구체성 | 수치/종목명/금액 포함 여부 | 0~3 |
| 시장 방향성 | 급락/폭등 등 강한 톤 키워드 | 0~3 |
| 신선도 | `exp(-0.005 × age_minutes)` — 0분=1.0, 1시간=0.74, 24시간≈0 | 0~1 |
| 속보 보너스 | Velocity Signal 감지 시 +10 | 0/10 |
| **최종** | `(영향도+신뢰도+구체성+방향성) × 신선도 + 속보보너스` | |

### 속보 감지 파이프라인

```
뉴스 수집 → 자카드 유사도 클러스터링 (≥0.3)
  → Velocity Signal: 동일 주제 3건+ / 다른 소스 / 15분 이내
    → 고영향 키워드: macro+geopolitical 2개+ & 60분 이내
      → breaking_score 계산 → 패스트패스 (≥15) 최상위 배치
```

### 2단계 LLM 브리핑 생성

![LLM 브리핑 플로우](docs/llm-flow.png)

```
[1차 호출] 매크로 전략가 프롬프트
  입력: 거시경제 데이터 + 뉴스 헤드라인
  출력: 시장 국면/리스크/기회/전망 (500자)

[2차 호출] 브리핑 작성자 프롬프트
  입력: 1차 분석 + 원본 데이터
  출력: 번호 리스트 10항목 브리핑 (500자)
  → 초과 시 축약 재요청
```

## 🚨 속보 전용 자동 게시 (Breaking News Flash Post)

score ≥ 20인 고영향 속보를 감지하면 정규 스케줄과 별개로 즉시 게시:

```
[BreakingNewsMonitor] 데몬 스레드 (10분 폴링)
  → fetch_news() 호출 → score ≥ 20 속보 필터
    → 자카드 유사도 중복 체크
      → 쿨다운 15분 체크 + 일일 3건 한도 체크
        → LLM으로 속보 전용 포스트 생성 (500자 제한)
          → Threads + Discord + Telegram 게시
```

### 환경변수

```env
DAILY_FLASH_LIMIT=3                 # 일일 속보 게시 한도
FLASH_POST_SCORE_THRESHOLD=20       # 속보 게시 점수 임계값
FLASH_POLL_INTERVAL_SECONDS=600     # 폴링 간격 (초)
FLASH_COOLDOWN_MINUTES=15           # 연속 게시 쿨다운 (분)
```

## 🇺🇸 트럼프 Truth Social 모니터

트럼프의 Truth Social 포스트를 실시간 모니터링하여, 한국 주식시장/글로벌 경제에 영향이 큰 발언을 감지하면 자동으로 포스트를 게시합니다.

```
[TrumpMonitor] 데몬 스레드 (30분 폴링)
  → CNN 아카이브 JSON에서 최신 글 수집 (Range 헤더로 앞쪽 64KB만 요청)
    → 마지막 확인 ID 이후 새 글만 필터링
      → LLM 영향도 판단 (HIGH / LOW)
        → HIGH: 시장 영향 분석 포스트 생성 → Threads + Discord + Telegram 게시
        → LOW: 스킵 (생일 축하, 정치 공격, 개인 자랑 등)
```

### 데이터 소스
- CNN이 5분마다 업데이트하는 트럼프 Truth Social 아카이브 JSON 사용
- `Range: bytes=0-65536` 헤더로 앞쪽 64KB만 요청 (최신 글 20~30개, RSS 피드 수준의 가벼운 요청)
- 첫 실행 시 최신 ID만 기록하고 기존 글에는 반응하지 않음

### LLM 영향도 판단 기준

| 판정 | 기준 | 예시 |
|------|------|------|
| HIGH | 관세/무역, 금리/연준, 전쟁/군사, 에너지/원유, 중국/한국 언급, 글로벌 공급망 | 이란 군사 공격 5일 연기, 중국 관세 25% 부과 |
| LOW | 생일 축하, 정치 공격, 선거 캠페인, 개인 자랑, 미국 내정 | "HAPPY BIRTHDAY", 민주당 비판, 시청률 자랑 |

### 제한
- 하루 최대 3건 포스팅
- 30분 폴링 간격
- 20자 미만 글 자동 스킵 (이미지만 올린 경우 등)

## 📡 뉴스 수집 시스템 상세 (6단계 파이프라인)

```
[1단계] RSS 피드 9개 소스 수집 (API 키 불필요, 기본 커버리지)
  국내증시: 매경증권, 한경금융, 이데일리증권, 연합뉴스경제, 머니투데이
  글로벌:   한경글로벌, 매경국제
  테마:     매경IT
  속보:     연합뉴스속보
  ※ 각 항목에서 발행 시각(published_parsed/updated_parsed) 추출 → KST 정규화

[2단계] 시장 데이터 기반 동적 키워드 자동 생성 (LLM 호출 없음)
  코스피/코스닥 ±1.5% 이상 → "코스피 급락/급등"
  달러원 ±0.8% 이상        → "환율 원달러"
  WTI ±2% 이상             → "유가 급등/급락"
  나스닥 ±1.5% 이상        → "나스닥 미국증시 하락/상승"

[3단계] 네이버 뉴스 API로 동적 키워드 + 고정 키워드 검색 (보강)
  30분 이내 뉴스에 realtime_breaking 카테고리 자동 부여

[4단계] 속보 감지 (Breaking News Detection)
  4-1. 주제 클러스터링: 자카드 유사도(≥0.3) 기반 Union-Find 알고리즘
  4-2. Velocity Signal: 동일 주제 3건+ / 서로 다른 소스 / 15분 이내 → 속보 판별
  4-3. 고영향 키워드: macro+geopolitical 키워드 2개+ & 60분 이내 → 속보 후보
  4-4. breaking_score 계산 → is_breaking 플래그 설정

[5단계] 통합 스코어링 + 신선도 감쇠
  ① 영향도 (impact_score 0~15+): 키워드 사전 매칭 + 동적 키워드 보너스
  ② 출처 신뢰도 (source_trust 1~3): 연합뉴스/한경=3, 매경/머니투데이=2, 기타=1
  ③ 구체성 (specificity_score 0~3): 수치/종목명/금액 포함 여부
  ④ 시장 방향성 (sentiment_score 0~3): 급락/폭등 등 강한 톤 키워드
  ⑤ 신선도 계수 (freshness_score 0.0~1.0): exp(-λ × age_minutes)
    0분=1.0 / 30분=0.86 / 1시간=0.74 / 6시간=0.16 / 24시간≈0
  ⑥ 속보 보너스 (breaking_bonus): 속보 감지 시 +10점
  ⑦ 최종 점수: final = (①+②+③+④) × ⑤ + ⑥

[6단계] 패스트패스 + 최종 선별
  breaking_score ≥ 15인 뉴스 → 최상위 강제 배치 (최대 2건)
  나머지 일반 뉴스 → final_score 내림차순 (동점 시 최신 우선)
  총 8건 선별
  → 🚨 패스트패스 / 🔴 고영향(4점+) / 🟡 중영향(2점+) / ⚪ 일반
```

### 뉴스 캐시 (속보 Velocity Signal용)
- `_recent_news_cache`: 최근 수집 뉴스 보관 (최대 200건, TTL 2시간)
- 날짜 변경 시 전체 초기화
- 파이프라인 실행마다 캐시 정리 후 신규 뉴스 추가

## 📰 Discord / Telegram 뉴스 헤드라인 전송

게시물이 실제로 올라가는 시점에 선별된 뉴스 헤드라인을 Discord Embed 카드 + Telegram 메시지로 동시 전송합니다.

```
[게시 시점 연동]
  모닝 브리핑 게시 → 상위 8건 뉴스 전송 (context: "모닝 브리핑")
  장 마감 리포트 게시 → 상위 8건 뉴스 전송 (context: "장 마감")
  장중 한마디 게시 → 상위 8건 뉴스 전송 (context: "한마디")
  속보 게시 → 해당 속보 1건만 전송 (context: "🚨 속보")
  트럼프 HIGH 판정 게시 → 해당 발언 1건 전송 (context: "🇺🇸 트럼프")
```

### Discord Embed 카드 포맷
```
📰 뉴스 헤드라인 브리핑
🚨 1. [속보 제목](링크)  출처  점수
🔴 2. [고영향 제목](링크)  출처  점수
🟡 3. [중영향 제목](링크)  출처  점수
⚪ 4. [일반 제목](링크)  출처  점수
...
────────────────────
2026-03-24 08:00 | 모닝 브리핑 | 🚨속보 1건 | 🔴고영향 3건 | 총 8건
```

| 이모지 | 기준 |
|--------|------|
| 🚨 | 속보 (fast_path) |
| 🔴 | 고영향 (score ≥ 4) |
| 🟡 | 중영향 (score ≥ 2) |

## 🛡️ Threads 스팸 필터 방지

Meta의 자동 스팸 감지 시스템이 API를 통한 반복적 게시 패턴을 감지하여 게시물을 삭제할 수 있습니다. 이를 방지하기 위해 다음 조치를 적용했습니다.

| 조치 | 내용 | 효과 |
|------|------|------|
| 링크 본문 통합 | 속보/트럼프 링크를 reply 분리 대신 본문 하단에 포함 | reply 0건, 게시 1건으로 통합 |
| 테마 댓글 딜레이 | 본문 게시 후 30~60초 랜덤 대기 후 테마 댓글 게시 | 사람 패턴에 가깝게 |
| 소수점 정수 변환 | 10.66 → 11 자동 변환 | 스팸 필터 트리거 방지 |
| 일일 게시 수 제한 | 브리핑 2건 + 한마디 5건 + 속보 3건 + 트럼프 3건 = 최대 ~15건 | 과다 게시 방지 |

## 📝 로그

| 파일 | 내용 |
|------|------|
| `bot.log` | 전체 봇 동작 로그 (RotatingFileHandler, 5MB 로테이션, 최대 3백업) |
| `output/YYYYMMDD_morning.txt` | 모닝 브리핑 원문 |
| `output/YYYYMMDD_closing.txt` | 장 마감 리포트 원문 |

### 리소스 관리
- `bot.log`: 5MB 초과 시 자동 로테이션 (bot.log.1, .2, .3 보관, 총 20MB 상한)
- CPU: schedule 30초 폴링 + threading.Timer(daemon) 기반, 유휴 시 부하 거의 없음

## 🤖 LLM 사용 방식

### 지원 모델

| Provider | 모델 | 용도 |
|----------|------|------|
| **Anthropic** | Claude Sonnet 4 | 시장 분석, 브리핑 생성, 속보 요약, 트럼프 영향도 판단 |
| **OpenAI** | GPT-4o | 동일 |
| **Google** | Gemini 2.0 Flash | 동일 |

### LLM 호출 지점

| 기능 | 호출 횟수/일 | 용도 |
|------|-------------|------|
| 모닝 브리핑 | 2~3회 | 정세 판단 + 브리핑 생성 + (축약) |
| 장 마감 리포트 | 2~3회 | 정세 판단 + 브리핑 생성 + (축약) |
| 장중 한마디 | 5회 | 한마디 생성 |
| 속보 자동 게시 | 0~3회 | 속보 요약 |
| 트럼프 모니터 | 10~20회 | 영향도 판단 (대부분 LOW) |
| **합계** | **~20~35회/일** | |

### 프롬프트 설계 원칙
- 역할 부여: "20년 경력 매크로 전략가", "주식시장 전문 브리핑 작성자"
- 데이터 기반: "제공된 데이터의 실제 수치만 사용"
- 포맷 강제: 번호 리스트 10항목, 각 25자 이내
- 길이 제한: 500자 이내 (초과 시 축약 재요청)
